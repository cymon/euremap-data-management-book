# 2. Data Management

## Preamble

**Here are a few key recommendations to keep in mind:**

- Create a Data Management Plan before beginning your project. The DMP is a
  evolving document and should be updated during the course of the project when
  deviations from the original plan are necessary.

- Deposit data sets in the most data-specific repository available and
  appropriate for your data type. For instance, DNA sequence data should be
  deposited in EMBL ENA rather than a general data repository such as Zenodo or
  Pangaea. Because EUREMAP is a Horizon Europe project of the European
  Commission we favour using data infrastructure that is part of the [European
  Open Science Cloud
  (EOSC)](https://research-and-innovation.ec.europa.eu/strategy/strategy-research-and-innovation/our-digital-future/open-science/european-open-science-cloud-eosc_en).
  
- Deposit data sets as soon after collection as is reasonably possible. Once you
  have collected and described your data (you have all the metadata), you should
  deposit it. This prinicple ensures that the data are not forgotten or lost,
  and they are deposited at the time when data collector is most conscious of
  their identity.

- If necessary, place an embargo on the date of release of the data set. This
  will not normally be necessary, but is an option. Most all repositories allow
  embargos on data release to be made, but often differ in whether the embargo
  is automatically or manually ended thereby publising the data.

## A EUREMAP Project Data Management Plan

The EUREMAP project uses the [OpenAIRE Argos](https://argos.openaire.eu/portal)
service to create and organise data management plans.

A DMP within Argos (called a *"Plan"*) consists of a *"Blueprint"* template of
generic information to which data set *"Descriptions"* are added as necessary to
describe all the data sets to be collected by the project.

**Procedure for creating a DMP in Argos:**

- Create an [account on Argos](https://argos.openaire.eu/portal) if necessary, and login.

- EUREMAP has a pre-formated project-specific *"Blueprint"*. To begin a new
  project Plan, click the *"Start New Plan"* button on the top-right of the
  frontpage. In the Search bar, search for *"EUREMAP Plan Blueprint"*, and
  select.

- Complete the fields in Sections 1 and 2.

- **For each data set (or data type) to be created**, select an appropriate
  "Description" in Section 3. The default should be "EUREMAP | generic | template
  Version 1". Complete the information required in the Description template, and
  repeat as necessary for each data set.

- On publishing your completed DMP, it will be accessioned within the Zenodo
  repository where it can be included in the [Zenodo EUREMAP
  Community](https://zenodo.org/communities/euremap/records?q=&l=list&p=1&s=10&sort=newest)
  space. DO NOT PUBLISH TO THE DEFAULT ARGOS COMMUNITY ON ZENODO!

- Several examples of Public Plans using the EUREMAP templates can be found by
  searching in Argos and Zenodo, e.g. [EUREMAP demonstrator project: Enhancing
  the Production and Therapeutic Potential of Streptocyclinones for Alzheimer’s
  Disease](https://argos.openaire.eu/explore-plans/overview/public/71a0d90d-8b3e-4f8d-81c1-e1d0ef950385)
